# 测试代码与正式代码边界整理

## 背景

- `AGENTS.md` 已新增规则：测试代码和正式运行代码必须分开，测试夹具、mock、fake、fixture、断言辅助、smoke/harness 入口不得混入 `src/`、`src-tauri/src/` 等生产路径。
- 当前主工作区已有大量未归因改动，并且 `lane-mcp-tools-only-server-refactor` 正在 active；本整理任务必须并行可见，优先避开其 shared paths。
- 本任务目标是持续把现有代码按该规则整理到位，不用一次大批量迁移替代可验证切片。

## 目标

- 盘点现有生产路径中疑似测试专用代码：`mock`、`fake`、`fixture`、`harness`、`testSupport`、smoke-only 入口、断言辅助和只被测试引用的工具。
- 按风险分批把测试专用能力迁到 `tests/`、`*.test.*`、`__tests__/` 或明确 test-support 目录/命名空间。
- 保留生产代码必要的可注入接口、稳定抽象和运行时实现；不为了测试迁移破坏现有 public 契约。
- 每个迁移切片都有相邻测试或构建验证，Round Log 记录实际路径、验证和残余风险。

## 非目标

- 不在本 lane 中重做 MCP tools-only server 架构；该范围归 `lane-mcp-tools-only-server-refactor`。
- 不宽泛格式化、重排或批量移动共享热点文件。
- 不删除仍被生产运行时依赖的抽象、adapter 或 facade；测试边界整理不等于去掉可测试设计。
- 不在未确认影响前迁移 settings、MCP、storage 等当前 active lane shared paths。

- 本 lane 初始 owned path 只包含本计划和盘点报告；代码 owned path 在 TASK-001 后按切片追加。
- 前端和 Rust 盘点可并行进行，但每个执行切片必须有不重叠写入路径。
