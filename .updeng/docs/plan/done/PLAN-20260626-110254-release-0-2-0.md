# 发布 0.2.0 版本

## 目标
- 将 Kerminal 发布为 `v0.2.0`。
- 同步 `package.json`、`src-tauri/Cargo.toml` 和 `src-tauri/tauri.conf.json` 三处版本。
- 运行发布前快速门禁后，在确认的 release commit 上创建 `v0.2.0` tag 并推送触发 GitHub Actions draft release。
- CI 产物完成后核对 GitHub Release assets、updater `latest.json` 和 `.sig`。

## 非目标
- 不生成或轮换 Tauri updater 签名密钥。
- 不手写 updater 签名、文件名或下载链接。
- 不在 CI 产物未完整核验前公开 draft release。

## 影响范围
- 版本文件：`package.json`、`package-lock.json`、`src-tauri/Cargo.toml`、`src-tauri/Cargo.lock`、`src-tauri/tauri.conf.json`。
- 发布配置：`.github/workflows/release.yml`。
- 外部副作用：`origin/main` push、`v0.2.0` tag push、GitHub Actions draft release。
- 流程台账：本计划、`.updeng/docs/plan/INDEX.md`、`.updeng/docs/in-progress.md`、`.updeng/docs/coordination/lanes.json`。

- Release workflow 会使用 tag 对应 commit 构建；tag 指错 commit 会导致版本和产物错位。
- Tauri updater 签名依赖 GitHub secrets；本地不读取、不生成、不记录密钥。
