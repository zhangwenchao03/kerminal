# SFTP 传输交互增强

## 目标
- 在右侧 SFTP 工具面板中补齐上传和下载入口，让用户可以填写本地路径与远程路径，并在执行前看到中文确认信息。
- 文件列表中的普通文件支持一键带入下载路径，降低手工输入远程路径的成本。
- 传输成功后给出明确状态；上传成功后刷新当前目录。

## 非目标
- 本次不接入原生文件选择器、拖拽上传、传输队列、断点续传或并发传输。
- 本次不修改 Rust SFTP 后端命令契约。
- 本次不改变 SFTP 删除、重命名、新建目录的后端能力。

## 影响范围
- 前端页面：`src/features/tool-panel/ToolPanel.tsx`
- 前端 API 测试/组件测试：`src/features/tool-panel/ToolPanel.test.tsx`
- 长期计划：`.updeng/docs/plan/next/terminal-product-plan.md`

## 风险
- 传输操作可能覆盖本地或远程同名文件，因此 UI 必须有二次确认。
- 本次仍使用手动路径输入，原生文件选择器留到后续更高风险切片。

## 结果
- 右侧 SFTP 工具面板新增“文件传输”区域，支持上传/下载模式切换、本地路径、远程路径、执行前确认和成功/错误状态。
- 文件列表中的普通文件和符号链接显示“下载”按钮，可把远程路径和默认 `~/Downloads/<文件名>` 本地路径带入表单。
- 上传成功后会刷新当前远程目录；下载成功后保留当前浏览位置。
