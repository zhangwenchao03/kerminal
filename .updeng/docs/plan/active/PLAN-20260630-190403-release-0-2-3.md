# 发布 0.2.3 版本

## 目标
- 将 Kerminal 发布为 `v0.2.3`。
- 发布刚提交的 PTY 稳定性加固与 Updeng 项目台账：`bf343097 feat(terminal): harden local PTY stability`。
- 同步桌面版本文件到 `0.2.3`，创建并推送 `v0.2.3` tag 触发 GitHub Actions release。
- CI 完成后核对 GitHub Release assets、updater `latest.json` 和 `.sig`。

## 非目标
- 不生成或轮换 Tauri updater 签名密钥。
- 不手写 updater 签名、文件名或下载链接。
- 不调整 GitHub Actions release 矩阵，除非它阻塞 `0.2.3` 发布。
- 不执行干净机器安装或真实应用内更新重启，除非本机环境已经具备。

## 风险
- Release tag 必须指向同步版本后的 release commit，不能指向功能提交。
- Tauri updater 签名依赖 GitHub secrets；本地不读取、不生成、不记录密钥。
- 本轮将推送 GitHub 远端并公开发布 Release，属于外部副作用；用户已明确要求“提交代码，发布下个版本”。
- 已发布版本的干净环境安装和应用内更新重启仍需要对应平台环境或人工验收。
