# 领域上下文文档格式

## 推荐路径

默认文件：

```text
.updeng/docs/domain/context.md
```

如果项目已有 `CONTEXT.md`、`docs/domain.md` 或业务目录约定，沿用现有位置，并在 Updeng 文档索引中链接。

## 文档结构

```markdown
# 领域上下文

## 术语

### <Canonical Term>

- 含义：
- 不等于：
- 常见同义词：
- 代码入口：
- 示例：

## 状态与生命周期

### <Entity>

| 状态 | 含义 | 进入条件 | 退出条件 |
| --- | --- | --- | --- |

## 关系与边界

- <概念 A> 与 <概念 B> 的关系：
- 谁拥有状态：
- 谁只读引用：

## 待确认

- <问题>：影响 <实现/测试/文档>
```

## 写入规则

- 只写领域语言和业务语义，不写实现计划。
- 可链接代码入口，但不要复制大段代码。
- 一个概念只保留一个 canonical term。
- 用户确认前放在“待确认”，确认后移入正式术语。
- 被 ADR 决定的技术取舍只引用 ADR，不在上下文文档重复解释。

## 质量检查

- 新术语是否能用于测试名、接口名、issue 标题。
- 是否明确“不等于什么”，避免误用。
- 是否说明状态变化和边界条件。
- 是否删除了临时讨论和重复段落。
