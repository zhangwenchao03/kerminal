# 业务文档规范与模板

用于 `.updeng/docs/biz/` 业务文档。项目事实写入 `.updeng/docs/biz/`，模板规则保留在本文件。

## 写作目标

- 人能快速理解业务目标、范围、规则、边界和风险。
- AI 能快速定位代码入口、数据入口、修改影响域和验证口径。
- 文档记录稳定事实，不复制大段代码，不把临时计划写成已完成结论。

## 信息边界

- 业务文档写项目特有业务规则、流程、行为边界、实现入口和验证口径。
- 长期技术/架构/流程取舍写 `.updeng/docs/decisions/`；待执行任务写 `.updeng/docs/plan/INDEX.md` 和对应状态目录，或写 `.updeng/docs/issues/`；机器验证证据写 `.updeng/docs/`。
- 代码入口、表名、权限标识、枚举、字段含义必须来自代码或可信项目文档；不能确认时写“待核对”，不要写成确定事实。
- 大型业务文档可保留三级标题，但二级标题尽量对齐标准结构。

## 新增文档规则

- 新增业务文档必须使用“标准模板”。
- 新增文档后同步更新 `.updeng/docs/biz/README.md`。

## 修改存量文档规则

- 每次修改某个业务域时，至少补齐与本次改动相关的快速索引、修改指南和验证清单。
- 已经有“目标、当前逻辑、方案与决策、后续事项、风险与注意事项”的文档，可以渐进映射：
  - `目标` -> `一句话说明` 和 `适用范围`
  - `当前逻辑` -> `核心流程`、`业务规则` 和 `数据规则`
  - `方案与决策` -> 业务规则留在本文；长期技术取舍迁移到 `.updeng/docs/decisions/`
  - `后续事项` -> 仍需执行的内容迁移到 `.updeng/docs/plan/INDEX.md`、对应计划状态目录或 `.updeng/docs/issues/`
  - `风险与注意事项` -> 保留并补充 `边界与异常`
- 补“快速索引”必须以代码或既有文档为证据；不确定的入口宁可留空并写明待核对。

## 阅读顺序

1. 先看“一句话说明、适用范围、快速索引”，确认是否命中当前业务域。
2. 再看“核心流程、业务规则、数据规则”，理解系统当前行为。
3. 修改代码前看“边界与异常、修改指南、验证清单、历史决策”。

## 标准模板

```markdown
# 业务能力名称

## 一句话说明

用 1 到 3 句话说明这个能力解决什么业务问题、最终形成什么业务结果。

## 适用范围

- 包含：本能力覆盖的业务场景。
- 不包含：容易混淆但不归本文负责的场景。
- 相关角色/权限：涉及的角色、权限标识或数据范围。

## 快速索引

| 类型 | 位置 | 说明 |
| --- | --- | --- |
| Controller | `类名#方法名` | 主要接口入口 |
| Service | `类名#方法名` | 核心业务逻辑 |
| Mapper | `类名#方法名` | 关键查询或写入 |
| 表/视图 | `table_name` | 关键数据来源 |
| 前端页面 | `path` | 如有则填写 |
| 权限/字典/定时任务 | `key` | 如有则填写 |

## 核心流程

1. 用户、定时任务或第三方系统触发什么动作。
2. 后端做哪些校验、查询、状态判断或权限过滤。
3. 系统写入或更新哪些关键数据。
4. 返回什么结果，或者通知哪些外部系统。
5. 失败、跳过或重试时怎么处理。

## 业务规则

| 场景 | 规则 | 影响代码/数据 |
| --- | --- | --- |
| 新增 | 必填、默认值、重复判断 | `xxxService#create` |
| 修改 | 哪些字段允许改，哪些状态禁止改 | `xxxService#update` |
| 删除/关闭 | 物理删除、逻辑删除、引用校验或归档 | `xxxService#delete` |

## 数据规则

- 关键字段：字段含义、单位、可空性、默认值。
- 状态枚举：状态码、业务含义、允许流转方向。
- 空间数据：坐标系、WKT/WKB、PostGIS 字段或范围查询口径。
- JSON/数组字段：结构、数据迁移影响和历史数据注意事项。

## 边界与异常

- 无数据时：
- 权限不足时：
- 参数非法时：
- 重复提交或并发时：
- 第三方失败或超时时：

## 修改指南

- 新增字段：通常需要同步 Entity、DTO/VO/BO、Mapper SQL、导入导出、前端类型和业务文档。
- 新增状态：通常需要同步枚举、查询筛选、流程流转、统计、前端文案和验证用例。
- 新增筛选条件：通常需要同步接口参数、分页查询、计数 SQL、权限过滤和前端表单。
- 新增外部动作：通常需要同步操作日志、幂等、失败重试、超时、验签或回调记录。

## 验证清单

- 正常新增、修改、查询、删除或关闭。
- 权限、行政区划和数据范围过滤。
- 空值、非法值、重复值和边界状态。
- 分页、排序、统计和导出。
- 关联数据存在和不存在。
- SQL、空间查询、第三方调用或定时任务结果。

## 历史决策

- 记录“为什么这么做”，尤其是代码里看不出来、但后续容易误改的业务决策。

## 待确认与后续

- 待确认事实：还不能写成确定业务规则的问题。
- 后续事项：只保留指向 `.updeng/docs/plan/INDEX.md`、具体计划文件或 `.updeng/docs/issues/` 的链接，不在业务文档里维护任务清单。

## 风险与注意事项

- 记录当前仍可能踩坑的业务、数据、调用方、联调或运维风险。
```
