# completed 记录规范与模板

用于 `.updeng/docs/completed.md`。本文维护写法和模板，具体已完成能力写入
`.updeng/docs/completed.md`。

## 写作目标

- 人能在 30 秒内判断这个能力是否已经可用、入口在哪里、有什么边界。
- AI 能稳定抽取状态、入口、验证证据、关联业务文档和后续动作。
- completed 只放稳定结果，不放执行过程、临时计划、调研笔记或大段验证日志。

## 记录粒度

- 一条记录对应一个对用户或后续开发有意义的稳定能力。
- 能力过大时拆成多条，例如“模型管理”“MCP 管理”“RAG 知识库”。
- 能力过小时合并到所属能力，例如单个字段调整、按钮文案调整、计划清理。
- 只调整 skill 文案、模板、格式或提示词，且没有形成可复用仓库能力时，通常不写入 completed。

## 字段规则

- `状态`：固定写 `已完成`，不要写进行中内容。
- `完成时间`：写 `YYYY-MM-DD`；无法确认时写“已完成，日期待核对”。
- `负责人/来源`：写需求来源、任务来源或实现者；不确定可写“项目迭代”。
- `验证`：写最近一次有效验证命令或人工证据；未验证必须写清原因。
- `关联文档`：优先链接 `.updeng/docs/biz/`，没有业务文档时链接 README、配置文档或代码入口。
- `后续动作`：只写稳定能力之外的明确后续项；没有就写“无”。

## 标准记录模板

```markdown
### 能力名称

| 字段 | 内容 |
| --- | --- |
| 状态 | 已完成 |
| 完成时间 | YYYY-MM-DD |
| 负责人/来源 | 用户需求 / 项目迭代 / 修复任务 |
| 验证 | `命令或人工验证证据` |
| 关联文档 | `.updeng/docs/biz/xxx.md` |

**能力说明**

- 一句话说明这个能力现在能做什么，以及对用户或后续开发有什么价值。

**使用入口**

| 类型 | 位置 | 说明 |
| --- | --- | --- |
| 后端接口 | `Controller#method` 或 `/api/path` | 主要调用入口 |
| 前端页面 | `ui/src/views/...` | 如有则填写 |
| 服务实现 | `ServiceImpl#method` | 核心逻辑入口 |
| 数据/配置 | `table_name` / `application.yml` | 如有则填写 |

**完成范围**

- 已完成的稳定行为 1。
- 已完成的稳定行为 2。

**边界与非目标**

- 当前不覆盖的场景、依赖前提或需要避免误读的边界。

**验证证据**

- `命令`：结果摘要。
- 人工验证：页面、接口、数据库或截图证据摘要。

**后续动作**

- 无。
```

## 示例记录

```markdown
### 管理端主题切换

| 字段 | 内容 |
| --- | --- |
| 状态 | 已完成 |
| 完成时间 | 2026-06-03 |
| 负责人/来源 | 用户体验增强 |
| 验证 | `cd ui; pnpm typecheck; pnpm lint; pnpm build` 通过 |
| 关联文档 | `.updeng/docs/biz/theme-toggle.md` |

**能力说明**

- 管理端已支持浅色、深色和跟随系统三种主题模式，并持久化用户选择。

**使用入口**

| 类型 | 位置 | 说明 |
| --- | --- | --- |
| 前端组件 | `ui/src/components/ThemeToggle.tsx` | 主题切换按钮 |
| 前端 Hook | `ui/src/hooks/useTheme.ts` | 主题状态、持久化和系统主题监听 |
| 布局入口 | `ui/src/layouts/DashboardLayout.tsx` | 侧边栏底部挂载入口 |
| 首屏脚本 | `ui/index.html` | 预设 `dark` class，减少闪烁 |

**完成范围**

- 用户可以在浅色、深色和跟随系统之间切换。
- 主题选择写入 `localStorage`，刷新后保持。
- 首屏加载前根据持久化选择或系统主题预设页面 class。

**边界与非目标**

- 不改变业务页面的数据结构和接口契约。
- 不负责重新设计整套视觉主题，只提供全局主题切换能力。

**验证证据**

- `cd ui; pnpm typecheck; pnpm lint; pnpm build`：通过。
- 浏览器人工验证：切换三种主题并刷新页面，主题状态保持。

**后续动作**

- 无。
```
